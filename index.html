Under the Hood

This site is more than just HTML. It evolved from a manually deployed setup into a fully automated Infrastructure-as-Code pipeline. Here is how the serverless backend operates:

User ➔ CloudFront ➔ S3 (Frontend) & API Gateway ➔ Lambda ➔ DynamoDB

Core Components

  • Frontend & CDN: Hosted securely in an S3 bucket with no direct public access, served globally and aggressively cached via CloudFront with enforced HTTPS.
  • Stateful Backend: A Python-based AWS Lambda function performs atomic read/increment operations on a pay-per-request DynamoDB table to track persistent visitor counts.
  • API Layer: API Gateway proxies requests to Lambda, configured with strict CORS policies to only allow browser access from this domain.
  • Automation (IaC): The entire stack is defined declaratively using Terraform, deployed automatically via GitHub Actions pipelines that invalidate cache on every push.

Engineering Challenges Solved

1. Terraform State & Git Bloat

While staging, Git tracked a large local AWS provider binary (~600MB+), breaking GitHub's 100MB limit. I solved this by resetting the Git index (`git rm -r --cached .`), executing a clean mixed reset, and rebuilding a clean commit history with proper `.gitignore` protocols.

2. Avoiding Redundant JSON Parsing

Initially, the frontend threw `SyntaxError: "undefined" is not valid JSON`. I realized HTTP API proxy integrations already parse JSON responses dynamically. Adjusting the fetch logic to utilize `response.json()` directly mapped the object, stabilizing the visitor counter.
