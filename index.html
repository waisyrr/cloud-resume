How I Built This Site

Cloud Resume Challenge Architecture

This portfolio is designed as a live technical project, not just a static resume page. The frontend is delivered through AWS, and the visitor counter uses a serverless backend.

Request Flow

Browser → CloudFront → S3 frontend
Browser → API Gateway → Lambda → DynamoDB → Lambda response → Browser update

Frontend Delivery

Static HTML, CSS, and JavaScript hosted in S3 and distributed globally with CloudFront. HTTPS and DNS are handled through the AWS edge and domain configuration.

Serverless Counter

A JavaScript fetch request calls API Gateway. Lambda increments and returns the persistent count stored in DynamoDB.

Infrastructure as Code

Terraform defines the infrastructure so the project is repeatable, documented, and easier to maintain than a manually configured cloud setup.

CI/CD Pipeline

Automated deployments are handled with GitHub Actions. Every push to the main branch synchronizes the website to Amazon S3 and invalidates the CloudFront cache, ensuring updates are published automatically.

Problems Solved

Terraform provider file issue

Cleaned up Git tracking after a large provider binary was added locally, then rebuilt the repository with better ignore rules.

Counter JSON parsing issue

Fixed frontend response handling so the counter uses the parsed JSON object returned by the API.
